快速安装Hugo

  • windows下载对应系统版本二进制文件：(https://github.com/gohugoio/hugo/releases)

    建议下载下图带extended的版本

    其他平台的安装方法参考官方文档：(https://gohugo.io/getting-started/installing/)

  • 配置Hugo的环境
  1. 解压下载好的二进制文件，目录如下

  1. 将hugo.exe程序放到配置环境的某个目录下

    例如我放在：D:\Hugo\Hugopath\bin\hugo.exe

    要放在你创建的bin目录下

  2. 将D:\Hugo\Hugopath\bin\hugo.exe路径保存到系统环境变量中

  1. 查看hugo是否配置成功 cmd中输入：

     hugo version

    返回版本号配置成功

初始化博客站点

  • 到创建的文件右键点击Git Bash Here打开Git输入:

    要安装Git才行(https://git-scm.com/)

    cmd中输入中也行，后面部署到github上需要Git

    hugo new site <博客站点目录名称>

      hugo new site blog

  • blog目录结构

- content：通常存放网站用到的md文档

- static：通常存放网站使用到的图片、音视频

- themes：网站使用到的主题

- config.toml：网站的配置文件

添加Hugo主题

  • Hugo主题(https://themes.gohugo.io/)
  1. 选择一个自己喜欢的主题，我这里用的是Dream。

  1. 点击Download转到(https://github.com/g1eny0ung/hugo-theme-dream)

  1. 将主题代码放置到themes目录下：
cd blog
git init
git submodule add git@github.com:g1eny0ung/hugo-theme-dream.git themes/hugo-theme-dream

  1. 将hugo-theme-dream主题目录下的config.toml

    覆盖初始config.toml

新建文章

hugo new posts/hello-hugo.md

该命令会在content/posts目录下生成hello-hugo.md文件，打开进行编辑：

draft: true表明该文章是草稿

如果在启用服务时不指定参数-D或–buildDrafts，或者在配置文件config.toml中配置buildDrafts = true，则会在生成文章时忽略草稿。

如果不想指定该参数就生成文章，需要改为draft: false或者将其删去。

启动服务并生成静态页面

  • 启动服务输入命令：

      hugo server -D

  • 生成静态页面：

      hugo -D

使用Git远程部署到github

  • 在GitHub上创建一个仓库

    首先在GitHub上创建一个仓库，仓库的名字格式为.github.io。比如我的GitHub用户名是yuanwine，那么这个仓库就命名为yuanwine.github.io。

  • SSH key的创建与配置

    因为要使用SSH的方式来和GitHub仓库进行交互，需要生成一对密钥对，然后将公钥配置到GitHub账号上。

    关于SSH key的创建与配置到GitHub自行百度下吧

  • 修改config.toml

    将config.toml文件中baseURL对应的网址的链接改成自己的博客链接

  • 将public中生成的静态文件部署到github上

cd public
git init
git add .
git commit -m "first-commit"
git branch -M main
git remote add origin git@github.com:yuanwine/yuanwine.github.io.git
git push -u origin main

  • 推送成功以后下次就不用带-u了

    每次推送时都要重新生成静态页面后再推送

hugo -D
cd public
git add .
git commit -m ""
git push origin main
